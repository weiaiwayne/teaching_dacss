Task 4: Save the data externally

save the dataframe as an .csv file

write.csv(influencer_posts, file="saved-influencer_posts.csv") ### save the new df as CSV files

save the dataframe or multiple files as an .rda file

save(influencer_posts,keyword_df, file="saved_posts.rda") ### save multiple files as a single .rda file - recommended!
load(file="saved_posts.rda") #use this code to load the saved .rda file

CSV files are more portable meaning that it can be opened by different programs (e.g., Google Spreadsheet, Excel, etc.). But you may encounter encoding problems when your csv files contain special or foreign characters, such as emojis. To prevent this, it is best to save it as a .rda file, which can be easily loaded into R.

Task 6: A visualization of top locations in user bios

An optional task for COMM students, but required for DACSS students.

library(ggplot2) 
library(ggthemes)

location_frequency <- as.data.frame(table(df$user_location))

# Rename columns
colnames(location_frequency) <- c("location", "count")

# Sort by frequency and select top 10 locations
top_10_locations <- head(location_frequency[order(-location_frequency$count), ], 10)

# Plot the top 10 locations
ggplot(top_10_locations, aes(x = reorder(location, -count), y = count)) +
  geom_bar(stat = "identity", fill = "steelblue") +
  labs(title = "Top 10 Locations in Twitter Bios", x = "Location", y = "Frequency") +
  theme_minimal() +
  theme(axis.text.x = element_text(angle = 45, hjust = 1))
